# KLIK_Line API

## `class Octoliner`

Создайте объект типа `Octoliner` для связи с конкретной платой Octoliner.

### `Octoliner(uint8_t i2cAddress = 42)`

Создает новый объект. Параметр `i2cAddress` задает I²C-адрес платы, который по умолчанию равен 42, 
но может быть изменен программно.

### `void begin()`
### `void begin(TwoWire* wire)`

Инициализирует заданный интерфейс, подготавливая плату к связи. Если аргумент `wire` опущен, 
соединение устанавливается по аппаратному интерфейсу I2C#0. Метод инициализирует указанный или
используемый по умолчанию интерфейс TwoWire.
Если он был инициализирован ранее, этот метод выполняет его повторную инициализацию.

Вызовите этот метод один раз в функции `setup()`.

### `void setSensitivity(uint8_t sense)`

Изменение значения сопротивления нагрузки для ИК-датчиков. Увеличение сопротивления нагрузки приводит 
к снижению чувствительности ИК-датчика.

Регулирует чувствительность ИК-фотоэлементов. Аргумент `sense` должен находиться в диапазоне [0; 255], 
однако эффективные значения начинаются примерно с 190. То есть используйте `190` для минимальной 
чувствительности и `255` для максимальной. Как правило: используйте более низкую чувствительность, 
если Octoliner находится близко к поверхности, и более высокую чувствительность, если он находится
на высоте нескольких миллиметров над поверхностью.

### `int16_t analogRead(uint8_t channel)`

Считывает сигнал яркости с заданного `канала`. Возвращает целое число в диапазоне [0; 4096] 
в качестве результата. Ожидается, что значение `канала` будет от `0` до `7`. 
Номера каналов выводятся на плату.

- Чем больше отраженного света (больше похоже на белый), тем ближе значение к `0`.

- Чем меньше отраженного света (больше похоже на черный), тем ближе значение к `4096`.

### `void analogReadAll(int16_t* analogValues)`

Функция считывает данные со всех 8 каналов в массив, на который указывает указатель `analogValues`. 
Длина массива должна быть достаточной для хранения 8 элементов.

### `virtual uint8_t mapAnalogToPattern(int16_t* analogValues) const`

Создает 8-битный шаблон из массива `analogValues`. Один бит соответствует одному каналу.
 `1` — для темного диапазона, `0` — для светлого. 
 Пороговое значение вычисляется автоматически в рамках алгоритма.

Переопределите этот метод, чтобы использовать собственный алгоритм. 

Например, используя другой пороговый уровень.

### `virtual float mapPatternToLine(int8_t pattern) const`

Интерпретация шаблона канала как положения линии в диапазоне от `-1.0` (на левом краю) до `+1.0` (на правом краю).

Когда линия находится ниже центра датчика, возвращаемое значение равно `0.0`.

Метод реализован таким образом, что подходит для отслеживания линии толщиной ~15–25 мм. 
Результат не является полностью аналоговым, а получается с шагом в ⅛. 
Тем не менее, возвращаемое значение является хорошим входным параметром для ПИД-регулятора.

Если текущие показания датчика не позволяют определить положение линии
(какой-то мусор под датчиком или линия сошла с пути), возвращается значение `NaN`.

Переопределите этот метод для работы с другими толщинами линий или условиями гонки.

### `uint8_t digitalReadAll()`

Считывает все 8 каналов и интерпретирует их как двоичный шаблон. 
Один бит соответствует одному каналу. «1» — для темного участка, «0» — для светлого.

### `float trackLine()`
### `float trackLine(uint8_t pattern)`
### `float trackLine(int16_t* analogValues)`

Метод оценивает положение линии под датчиком и возвращает значение в диапазоне 
от `-1.0` (в левом крайнем случае) до `+1.0` (в правом крайнем случае). 
Если линия находится под центром датчика, возвращаемое значение равно `0.0`.

Если аргумент не указан, метод считывает все каналы. В противном случае, 
предоставленные данные используются в качестве входных данных. 
Последний вариант полезен, если вы используете данные и для других целей 
(например, для визуализации на дисплее), поскольку позволяет избежать повторного считывания.

Если текущие показания датчика не позволяют определить положение линии 
(например, мусор под датчиком или смещение с траектории), 
возвращается предыдущая успешная оценка (которая кэшируется). Начальное значение равно `0.0`.

### `bool optimizeSensitivityOnBlack()`

Выполняет автоматическую настройку чувствительности. 
Перед оптимизацией разместите Octoliner над трассой таким образом, 
чтобы _все_ его датчики были направлены на черную линию/поле. 
Если это физически невозможно, направьте все датчики в пустоту. 
Алгоритм автоматически отрегулирует чувствительность в соответствии с текущим уровнем черного цвета.

Метод возвращает `true` в случае успеха. Если калибровка не удалась, 
он возвращает `false`, например, если датчики расположены над контрастной черно-белой 
или полностью белой поверхностью. В случае неудачи предыдущий уровень чувствительности остается неизменным.

Чтобы получить установленную чувствительность, используйте `getSensitivity`.

Время измерения зависит от уровня черноты. 
Типичное значение составляет менее 1 секунды, максимальное — 3 секунды.

### `uint8_t getSensitivity() const`

Возвращает текущий уровень чувствительности, 
ранее установленный с помощью `setSensitivity` или `optimizeSensitivityOnBlack`.

### `void changeAddress(uint8_t newAddress)`

Изменяет I²C-адрес модуля. Изменение вступает в силу только при включенной плате.
 Если вы хотите сохранить его навсегда, вызовите метод `saveAddress`.

### `void saveAddress()`

Сохраняет текущий I²C-адрес платы навсегда.
